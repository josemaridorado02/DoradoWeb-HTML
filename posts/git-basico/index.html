Guía Básica de Git para Principiantes

20 de enero de 2025

Introducción a Git

Git es un sistema de control de versiones distribuido que permite gestionar y rastrear cambios en proyectos de desarrollo de software. Es una herramienta fundamental para cualquier programador.

Conceptos Básicos

¿Qué es un repositorio?

Un repositorio es el lugar donde Git almacena todos los archivos y el historial completo de cambios de un proyecto. Puede ser local (en tu ordenador) o remoto (en servicios como GitHub o GitLab).

Estados de los archivos

Los archivos en Git pueden estar en tres estados principales:

  • Modified: El archivo ha sido modificado pero no guardado en el repositorio
  • Staged: El archivo modificado está marcado para incluirse en el próximo commit
  • Committed: Los datos están guardados de forma segura en el repositorio local

Comandos Esenciales

Inicializar un repositorio

git init

Este comando crea un nuevo repositorio Git en el directorio actual.

Clonar un repositorio existente

git clone https://github.com/usuario/repositorio.git

Descarga una copia completa de un repositorio remoto.

Verificar el estado

git status

Muestra qué archivos han sido modificados, añadidos o están listos para commit.

Añadir archivos al staging

# Añadir un archivo específico
git add archivo.txt

# Añadir todos los archivos modificados
git add .

Hacer un commit

git commit -m "Mensaje descriptivo del cambio"

Guarda los cambios del staging area en el historial del repositorio.

Ver el historial

git log

Muestra el historial de commits con sus mensajes, autores y fechas.

Trabajar con Ramas

Las ramas permiten desarrollar características de forma aislada sin afectar al código principal.

Crear una nueva rama

git branch nombre-rama

Cambiar de rama

git checkout nombre-rama

Crear y cambiar a una nueva rama

git checkout -b nueva-rama

Fusionar ramas

# Primero cambiar a la rama destino
git checkout main

# Luego fusionar
git merge nombre-rama

Trabajar con Repositorios Remotos

Añadir un remoto

git remote add origin https://github.com/usuario/repositorio.git

Subir cambios

git push origin main

Descargar cambios

git pull origin main

Buenas Prácticas

  1. Commits frecuentes y descriptivos: Realiza commits pequeños y con mensajes claros
  2. Usa ramas: Desarrolla nuevas características en ramas separadas
  3. Actualiza regularmente: Mantén tu repositorio local sincronizado con el remoto
  4. Revisa antes de commitear: Usa git status y git diff para verificar los cambios

Conclusión

Git es una herramienta poderosa que facilita la colaboración y el seguimiento de cambios en proyectos de software. Dominar estos comandos básicos es el primer paso para trabajar eficientemente en desarrollo de software moderno.

En futuros artículos exploraremos conceptos más avanzados como rebase, cherry-pick y estrategias de branching.

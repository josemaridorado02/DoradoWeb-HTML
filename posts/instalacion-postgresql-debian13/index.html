Instalación de PostgreSQL en Debian 13

30 de enero de 2025

Introducción

PostgreSQL es un sistema gestor de bases de datos relacional de código abierto, conocido por su robustez, cumplimiento de estándares SQL y capacidades avanzadas. Es ampliamente utilizado en aplicaciones empresariales que requieren integridad de datos, consultas complejas y características avanzadas como tipos de datos personalizados, índices especializados y soporte para JSON.

Instalación del Servidor

Actualizar el Sistema

sudo apt update

Instalar PostgreSQL

sudo apt install postgresql postgresql-contrib -y

El paquete postgresql-contrib incluye módulos y extensiones adicionales útiles.

Habilitar e Iniciar el Servicio

sudo systemctl enable postgresql
sudo systemctl start postgresql

Verificar el estado:

sudo systemctl status postgresql

Configuración Inicial

Acceder como Usuario postgres

PostgreSQL crea automáticamente un usuario del sistema llamado postgres:

sudo -i -u postgres

Entrar a PostgreSQL

psql

Crear Usuario y Base de Datos

-- Crear usuario
CREATE USER usuario WITH PASSWORD 'contraseña_segura';

-- Crear base de datos
CREATE DATABASE basedatos OWNER usuario;

-- Otorgar privilegios
GRANT ALL PRIVILEGES ON DATABASE basedatos TO usuario;

Salir de PostgreSQL

\q

Y salir del usuario postgres:

exit

Configuración para Acceso Remoto

PostgreSQL requiere configurar dos archivos para permitir conexiones remotas.

Configurar postgresql.conf

Este archivo controla qué interfaces escucha PostgreSQL:

sudo nano /etc/postgresql/17/main/postgresql.conf

Busca la línea listen_addresses (puedes usar Ctrl+W para buscar):

El asterisco * significa que escuchará en todas las interfaces de red disponibles.

Configurar pg_hba.conf

Este archivo controla la autenticación de clientes:

sudo nano /etc/postgresql/17/main/pg_hba.conf

Al final del archivo, añade esta línea:

Explicación:

  • host: Conexión TCP/IP
  • all (base de datos): Todas las bases de datos
  • all (usuario): Todos los usuarios
  • 0.0.0.0/0: Cualquier dirección IP
  • md5: Requiere contraseña encriptada con MD5

Configuración del Cliente

Instalar el Cliente PostgreSQL

sudo apt update
sudo apt install postgresql-client -y

Conectar Remotamente

psql -h IP_DEL_SERVIDOR -U usuario -d basedatos

Ejemplo:

psql -h 192.168.1.100 -U usuario -d basedatos

Se te pedirá la contraseña del usuario.

Ejemplo Práctico: Sistema de Biblioteca

Conectar a la Base de Datos

psql -h IP_DEL_SERVIDOR -U usuario -d basedatos

Crear Tablas

-- Tabla Autores
CREATE TABLE autores (
    id_autor INTEGER PRIMARY KEY,
    nombre VARCHAR(100) NOT NULL,
    nacionalidad VARCHAR(50),
    fecha_nacimiento DATE
);

-- Tabla Libros
CREATE TABLE libros (
    id_libro INTEGER PRIMARY KEY,
    titulo VARCHAR(200) NOT NULL,
    id_autor INTEGER NOT NULL,
    ano_publicacion INTEGER,
    genero VARCHAR(50),
    CONSTRAINT fk_libros_autores FOREIGN KEY (id_autor) 
        REFERENCES autores(id_autor)
);

-- Tabla Prestamos
CREATE TABLE prestamos (
    id_prestamo INTEGER PRIMARY KEY,
    id_libro INTEGER NOT NULL,
    nombre_usuario VARCHAR(100) NOT NULL,
    fecha_prestamo DATE DEFAULT CURRENT_DATE,
    fecha_devolucion DATE,
    CONSTRAINT fk_prestamos_libros FOREIGN KEY (id_libro) 
        REFERENCES libros(id_libro)
);

Comandos Útiles de PostgreSQL

Comandos Meta (con )

\l              -- Listar bases de datos
\c database     -- Conectar a una base de datos
\dt             -- Listar tablas
\d tabla        -- Describir estructura de tabla
\du             -- Listar usuarios
\q              -- Salir

Gestión de Bases de Datos

-- Crear base de datos
CREATE DATABASE nueva_bd;

-- Eliminar base de datos
DROP DATABASE nombre_bd;

-- Ver tamaño de base de datos
SELECT pg_size_pretty(pg_database_size('basedatos'));

-- Listar todas las bases de datos con su tamaño
SELECT 
    datname AS database,
    pg_size_pretty(pg_database_size(datname)) AS size
FROM pg_database
ORDER BY pg_database_size(datname) DESC;

Gestión de Usuarios y Privilegios

Crear Usuario

CREATE USER nuevo_usuario WITH PASSWORD 'password';

Usuario con Privilegios Específicos

-- Usuario solo lectura
CREATE USER lector WITH PASSWORD 'password';
GRANT CONNECT ON DATABASE basedatos TO lector;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO lector;

-- Usuario con permisos de escritura
CREATE USER editor WITH PASSWORD 'password';
GRANT ALL PRIVILEGES ON DATABASE basedatos TO editor;

Ver Privilegios

\du                          -- Lista usuarios y roles
\dp nombre_tabla             -- Privilegios sobre una tabla

Modificar Usuario

-- Cambiar contraseña
ALTER USER usuario WITH PASSWORD 'nueva_contraseña';

-- Hacer superusuario
ALTER USER usuario WITH SUPERUSER;

-- Permitir crear bases de datos
ALTER USER usuario WITH CREATEDB;

Eliminar Usuario

DROP USER nombre_usuario;

Copias de Seguridad

Exportar Base de Datos

pg_dump -h localhost -U usuario -d basedatos > backup.sql

Exportar con compresión:

pg_dump -h localhost -U usuario -d basedatos | gzip > backup.sql.gz

Importar Base de Datos

psql -h localhost -U usuario -d basedatos < backup.sql

Con compresión:

gunzip -c backup.sql.gz | psql -h localhost -U usuario -d basedatos

Backup de Todo el Servidor

pg_dumpall -U postgres > backup_completo.sql

Características Avanzadas de PostgreSQL

Soporte JSON

PostgreSQL tiene excelente soporte para JSON:

-- Crear tabla con columna JSON
CREATE TABLE productos (
    id SERIAL PRIMARY KEY,
    nombre VARCHAR(100),
    detalles JSONB
);

-- Insertar datos JSON
INSERT INTO productos (nombre, detalles) VALUES
('Laptop', '{"marca": "Dell", "ram": 16, "disco": "512GB SSD"}');

-- Consultar datos JSON
SELECT nombre, detalles->>'marca' AS marca
FROM productos
WHERE detalles->>'ram' = '16';

Funciones de Ventana

-- Ranking de libros por año
SELECT 
    titulo,
    ano_publicacion,
    RANK() OVER (ORDER BY ano_publicacion DESC) AS ranking
FROM libros;

Índices Especializados

-- Índice B-tree (por defecto)
CREATE INDEX idx_libros_titulo ON libros(titulo);

-- Índice para búsqueda de texto completo
CREATE INDEX idx_libros_titulo_ft ON libros USING GIN(to_tsvector('spanish', titulo));

-- Índice parcial
CREATE INDEX idx_prestamos_activos ON prestamos(id_libro) 
WHERE fecha_devolucion IS NULL;

Optimización y Mantenimiento

Analizar Tablas

ANALYZE autores;
ANALYZE libros;
ANALYZE prestamos;

Vacuum (Limpiar Espacio)

VACUUM autores;
VACUUM FULL;  -- Más agresivo, requiere bloqueo exclusivo

Ver Estadísticas

-- Tamaño de las tablas
SELECT 
    schemaname,
    tablename,
    pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS size
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY pg_total_relation_size(schemaname||'.'||tablename) DESC;

Solución de Problemas Comunes

No Puedo Conectar Remotamente

  1. Verifica listen_addresses = '*' en postgresql.conf
  2. Verifica la línea en pg_hba.conf: host all all 0.0.0.0/0 md5
  3. Reinicia PostgreSQL: sudo systemctl restart postgresql
  4. Verifica el firewall: sudo ufw allow 5432/tcp

Error “password authentication failed”

Verifica que el método de autenticación en pg_hba.conf sea md5 o scram-sha-256, no peer.

Error “FATAL: database does not exist”

Asegúrate de que la base de datos existe:

\l

Si no existe, créala:

CREATE DATABASE nombre_bd;

Conclusión

PostgreSQL es una de las bases de datos relacionales más avanzadas y confiables disponibles. Su cumplimiento de estándares SQL, características avanzadas como soporte JSON nativo, funciones de ventana, y extensiones personalizadas lo hacen ideal para aplicaciones complejas que requieren integridad de datos y consultas sofisticadas.

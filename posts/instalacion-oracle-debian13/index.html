Instalación de Oracle Database 21c en Debian 13

30 de enero de 2025

Introducción

Oracle Database es uno de los sistemas gestores de bases de datos relacionales más potentes y utilizados en entornos empresariales. En este tutorial aprenderemos a instalar Oracle Database 21c Express Edition en Debian 13 y configurarlo para acceso remoto.

Preparación del Sistema

Instalación de Dependencias

Primero actualizamos el sistema e instalamos las dependencias necesarias:

sudo apt update
sudo apt install rlwrap libaio1t64 libaio-dev unixodbc wget -y

Nota importante sobre Debian 13: En versiones anteriores se utilizaba el paquete libaio1, pero en Debian 13 este ha sido renombrado a libaio1t64.

Crear Enlace Simbólico para Compatibilidad

Oracle busca la librería /usr/lib/x86_64-linux-gnu/libaio.so.1, pero en Debian 13 se llama libaio.so.1t64. Creamos un enlace simbólico para mantener la compatibilidad:

sudo ln -s /usr/lib/x86_64-linux-gnu/libaio.so.1t64 \
           /usr/lib/x86_64-linux-gnu/libaio.so.1

Descarga de Oracle Database

Descargamos el paquete de instalación desde el sitio oficial de Oracle:

wget https://download.oracle.com/otn-pub/otn_software/db-express/oracle-database-xe-21c-1.0-1.ol8.x86_64.rpm

Conversión del Paquete RPM a DEB

Oracle distribuye sus paquetes en formato RPM (Red Hat Package Manager), pero necesitamos convertirlo a formato DEB para Debian.

Instalar Alien

Alien es una herramienta que permite convertir paquetes entre diferentes formatos:

sudo apt-get install alien

Alien soporta conversión entre formatos como RPM, DEB, SLP y otros.

Convertir el Paquete

sudo alien --scripts --to-deb oracle-database-xe-21c-1.0-1.ol8.x86_64.rpm

Advertencia: Este proceso puede tardar varios minutos. Una vez completado, encontrarás un archivo .deb en el mismo directorio.

Instalación de Oracle Database

Instalamos el paquete DEB generado:

sudo dpkg -i oracle-database-ee-21c_1.0-2_amd64.deb

Configuración del Sistema

Configurar Resolución de Nombres

Oracle necesita resolver correctamente el hostname. Añadimos la IP de nuestra máquina al archivo /etc/hosts:

echo "$(hostname -I | awk '{print $1}') $(hostname)" | sudo tee -a /etc/hosts

Esto previene problemas al arrancar el listener de Oracle.

Configurar el Servicio de Base de Datos

Ejecutamos el script de configuración que se instala con Oracle:

sudo /etc/init.d/oracledb_ORCLCDB-21c configure

Este script configura el servicio para que se inicie automáticamente con el sistema.

Permisos de Usuario

Añadimos nuestro usuario al grupo dba para tener privilegios de administración:

sudo usermod -aG dba $USER

Importante: Debes cerrar sesión y volver a iniciarla para que los cambios de grupo surtan efecto.

Variables de Entorno

Oracle requiere configurar varias variables de entorno. Las añadimos a nuestro archivo .bashrc:

echo 'export ORACLE_HOME=/opt/oracle/product/21c/dbhome_1' >> ~/.bashrc
echo 'export ORACLE_SID=ORCLCDB' >> ~/.bashrc
echo 'export NLS_LANG=SPANISH_SPAIN.AL32UTF8' >> ~/.bashrc
echo 'export ORACLE_BASE=/opt/oracle' >> ~/.bashrc
echo 'export LD_LIBRARY_PATH=$ORACLE_HOME/lib:$LD_LIBRARY_PATH' >> ~/.bashrc
echo 'export PATH=$ORACLE_HOME/bin:$PATH' >> ~/.bashrc
echo "alias sqlplus='rlwrap sqlplus'" >> ~/.bashrc
source ~/.bashrc

Explicación de variables:

  • ORACLE_HOME: Directorio de instalación de Oracle
  • ORACLE_SID: Identificador de la instancia de base de datos
  • NLS_LANG: Configuración de idioma y codificación
  • LD_LIBRARY_PATH: Rutas de las librerías compartidas
  • Alias sqlplus: Añade funcionalidad de edición de línea con rlwrap

Primer Acceso a Oracle

Conectamos como administrador:

sqlplus / as sysdba

Habilitar Creación de Usuarios

Por defecto, Oracle 21c requiere habilitar un parámetro especial para crear usuarios tradicionales:

ALTER SESSION SET "_ORACLE_SCRIPT"=true;

Crear Usuario y Asignar Privilegios

CREATE USER oteo IDENTIFIED BY 1234;
GRANT ALL PRIVILEGES TO oteo;

Recomendación de seguridad: En producción, utiliza contraseñas robustas y asigna solo los privilegios necesarios.

Configuración para Acceso Remoto

Configurar el Listener

Por defecto, Oracle solo escucha conexiones en localhost. Modificamos el archivo listener.ora:

sudo nano /opt/oracle/homes/OraDBHome21cEE/network/admin/listener.ora

Cambia la línea que contiene HOST = debian (o el nombre de tu host) por:

Esto permite que el listener acepte conexiones desde cualquier interfaz de red.

Configurar TNS Names

También debemos modificar tnsnames.ora para permitir conexiones remotas:

sudo nano /opt/oracle/homes/OraDBHome21cEE/network/admin/tnsnames.ora

Realiza el mismo cambio: reemplaza el hostname por 0.0.0.0.

Aplicar Cambios

Reinicia el servicio para aplicar la configuración:

sudo systemctl restart oracledb_ORCLCDB-21c.service

Configuración del Cliente

Instalar Dependencias en el Cliente

En la máquina cliente, instalamos los paquetes necesarios:

sudo apt update
sudo apt install rlwrap libaio1t64 libaio-dev wget p7zip-full -y

Crear Enlace Simbólico

Al igual que en el servidor:

sudo ln -s /usr/lib/x86_64-linux-gnu/libaio.so.1t64 \
           /usr/lib/x86_64-linux-gnu/libaio.so.1

Descargar Oracle Instant Client

Creamos un directorio para Oracle y descargamos los componentes necesarios:

mkdir ~/oracle
cd ~/oracle

wget https://download.oracle.com/otn_software/linux/instantclient/2119000/instantclient-basic-linux.x64-21.19.0.0.0dbru.zip

wget https://download.oracle.com/otn_software/linux/instantclient/2119000/instantclient-sqlplus-linux.x64-21.19.0.0.0dbru.zip

Extraer los Archivos

7z x instantclient-basic-linux.x64-21.19.0.0.0dbru.zip
7z x instantclient-sqlplus-linux.x64-21.19.0.0.0dbru.zip

Configurar Variables de Entorno

echo 'export ORACLE_HOME=$HOME/oracle/instantclient_21_19' >> ~/.bashrc
echo 'export PATH=$ORACLE_HOME:$PATH' >> ~/.bashrc
echo 'export LD_LIBRARY_PATH=$ORACLE_HOME:$LD_LIBRARY_PATH' >> ~/.bashrc
echo "alias sqlplus='rlwrap sqlplus'" >> ~/.bashrc
source ~/.bashrc

Conectar Remotamente

Desde el cliente, nos conectamos al servidor Oracle:

sqlplus usuario/password@//IP_DEL_SERVIDOR:1521/ORCLCDB

Ejemplo:

sqlplus oteo/1234@//192.168.1.100:1521/ORCLCDB

Crear Tablas

-- Tabla Autores
CREATE TABLE AUTORES (
    ID_AUTOR NUMBER PRIMARY KEY,
    NOMBRE VARCHAR2(100) NOT NULL,
    NACIONALIDAD VARCHAR2(50),
    FECHA_NACIMIENTO DATE
);

-- Tabla Libros
CREATE TABLE LIBROS (
    ID_LIBRO NUMBER PRIMARY KEY,
    TITULO VARCHAR2(200) NOT NULL,
    ID_AUTOR NUMBER NOT NULL,
    ANO_PUBLICACION NUMBER(4),
    GENERO VARCHAR2(50),
    CONSTRAINT FK_LIBROS_AUTORES FOREIGN KEY (ID_AUTOR) 
        REFERENCES AUTORES(ID_AUTOR)
);

-- Tabla Prestamos
CREATE TABLE PRESTAMOS (
    ID_PRESTAMO NUMBER PRIMARY KEY,
    ID_LIBRO NUMBER NOT NULL,
    NOMBRE_USUARIO VARCHAR2(100) NOT NULL,
    FECHA_PRESTAMO DATE DEFAULT SYSDATE,
    FECHA_DEVOLUCION DATE,
    CONSTRAINT FK_PRESTAMOS_LIBROS FOREIGN KEY (ID_LIBRO) 
        REFERENCES LIBROS(ID_LIBRO)
);

Solución de Problemas Comunes

Error al Iniciar el Listener

Si el listener no inicia, verifica que el hostname esté correctamente configurado en /etc/hosts.

Problemas de Permisos

Asegúrate de que tu usuario esté en el grupo dba y hayas cerrado/iniciado sesión.

Oracle no Arranca Automáticamente

Verifica que la tarea cron esté correctamente configurada con sudo crontab -l.

Conclusión

Oracle Database es una solución robusta para entornos empresariales que requieren alta disponibilidad, seguridad avanzada y características empresariales como particionamiento, RAC y Data Guard. Aunque su instalación y configuración inicial puede parecer compleja, siguiendo estos pasos tendrás un servidor Oracle completamente funcional en Debian 13.
